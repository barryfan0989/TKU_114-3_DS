# 測試案例說明文件 (Test Cases)

本文件列出針對 `ProductManagementSystem.java` 進行的 12 組測試案例，包含測試目的、輸入步驟與預期輸出結果，以確保系統的邏輯合法性與健全性。

---

## 測試案例列表

### 測試案例 1：顯示全部商品（初始狀態）
- **測試目的**：驗證系統在啟動後，能否正確載入並顯示 5 項預設商品。
- **輸入動作**：選擇選單項目 `1`。
- **預期輸出**：
  ```text
  -------------------------------------------
  編號     商品名稱            價格 (元)      庫存 (個)
  -------------------------------------------
  1        Keyboard          890          12      
  2        Mouse             490          20      
  3        Monitor           5200         5       
  4        USB Cable         250          30      
  5        Headset           1290         8       
  -------------------------------------------
  ```

### 測試案例 2：依完整名稱搜尋（忽略大小寫、忽略前後空白）
- **測試目的**：驗證搜尋功能是否可正確處理頭尾多餘空白以及大小寫不一致的輸入。
- **輸入動作**：選擇選單項目 `2`，輸入 `  mOuSe  `（前後有空白，大小寫混合）。
- **預期輸出**：
  ```text
  找到商品：Mouse，價格：490，庫存：20
  ```

### 測試案例 3：搜尋不存在之商品
- **測試目的**：驗證當搜尋不存在的商品名稱時，系統能給出正確的錯誤提示。
- **輸入動作**：選擇選單項目 `2`，輸入 `Laptop`。
- **預期輸出**：
  ```text
  找不到此商品！
  ```

### 測試案例 4：新增商品（正常流程）
- **測試目的**：驗證在有空位時能否成功新增商品，且資料能正確寫入陣列。
- **輸入動作**：
  1. 選擇選單項目 `3`。
  2. 輸入商品名稱：`Laptop`。
  3. 輸入價格：`25000`。
  4. 輸入庫存：`10`。
- **預期輸出**：
  ```text
  商品新增成功：Laptop，價格：25000，庫存：10
  ```
  *(再次執行選單 1 時，應能看到編號 6 為 Laptop 的商品)*

### 測試案例 5：新增商品（重複名稱阻擋）
- **測試目的**：驗證系統是否能正確阻擋重複名稱的商品新增。
- **輸入動作**：
  1. 選擇選單項目 `3`。
  2. 輸入商品名稱：`Keyboard`（已存在的商品）。
- **預期輸出**：
  ```text
  錯誤：商品名稱已存在，不可重複新增！
  ```
  *(系統不應繼續要求輸入價格與庫存，直接返回主選單)*

### 測試案例 6：出售商品（正常扣除庫存）
- **測試目的**：驗證正常購買時，庫存能正確扣除，且能記錄於本日銷售摘要中。
- **輸入動作**：
  1. 選擇選單項目 `4`。
  2. 輸入商品名稱：`Keyboard`。
  3. 輸入數量：`2`。
- **預期輸出**：
  ```text
  目前選中：Keyboard，價格：890，庫存：12
  出售成功！
  ```
  *(此時 Keyboard 庫存應降為 10)*

### 測試案例 7：出售商品（庫存不足阻擋）
- **測試目的**：驗證當出售數量大於當前庫存時，交易會被阻擋且庫存不被修改。
- **輸入動作**：
  1. 選擇選單項目 `4`。
  2. 輸入商品名稱：`Monitor`。
  3. 輸入數量：`100`（高於現有庫存 5）。
- **預期輸出**：
  ```text
  目前選中：Monitor，價格：5200，庫存：5
  出售失敗：數量錯誤或庫存不足！
  ```
  *(Monitor 的庫存應維持 5)*

### 測試案例 8：補充庫存（增加庫存）
- **測試目的**：驗證補貨邏輯能正確增加商品庫存量。
- **輸入動作**：
  1. 選擇選單項目 `5`。
  2. 輸入商品名稱：`Monitor`。
  3. 輸入數量：`10`。
- **預期輸出**：
  ```text
  目前選中：Monitor，價格：5200，庫存：5
  補貨成功！最新資料：Monitor，價格：5200，庫存：15
  ```

### 測試案例 9：修改商品價格（合法價格）
- **測試目的**：驗證透過 setter 方法是否能成功更新商品價格。
- **輸入動作**：
  1. 選擇選單項目 `6`。
  2. 輸入商品名稱：`USB Cable`。
  3. 輸入新價格：`280`。
- **預期輸出**：
  ```text
  目前選中：USB Cable，價格：250，庫存：30
  修改成功！最新資料：USB Cable，價格：280，庫存：30
  ```

### 測試案例 10：修改商品價格（非法價格阻擋）
- **測試目的**：驗證當輸入小於或等於 0 的價格時，修改會被阻擋，價格不被修改。
- **輸入動作**：
  1. 選擇選單項目 `6`。
  2. 輸入商品名稱：`USB Cable`。
  3. 輸入新價格：`-50`。
- **預期輸出**：
  ```text
  目前選中：USB Cable，價格：280，庫存：30
  修改失敗：價格必須大於 0！
  ```
  *(USB Cable 的價格應維持 280)*

### 測試案例 11：顯示低庫存商品
- **測試目的**：驗證系統能篩選出庫存少於 10 個的商品。
- **輸入動作**：選擇選單項目 `7`。
- **預期輸出**：
  ```text
  --- 低庫存商品 (庫存 < 10) ---
  編號     商品名稱            價格 (元)      庫存 (個)
  -------------------------------------------
  3        Monitor           5200         5       
  5        Headset           1290         8       
  -------------------------------------------
  ```

### 測試案例 12：結束並顯示操作摘要
- **測試目的**：驗證當結束系統時，能正確彙整並顯示當前 session 的所有銷售記錄及總金額。
- **輸入動作**：選擇選單項目 `0`（在已成功出售 2 個 Keyboard 的情況下）。
- **預期輸出**：
  ```text
  ===========================================
                 本日銷售摘要                
  ===========================================
  商品名稱            出售數量       銷售額 (元)  
  -------------------------------------------
  Keyboard          2            1780      
  -------------------------------------------
  本次銷售總金額：1780 元
  ===========================================
  
  系統已成功結束，感謝您的使用！
  ```
